# Google AdSense 合规指南

本项目目标是**通过 Google AdSense 审核并实现广告变现**。以下规则在开发中必须遵守。

## 硬性资格要求

- 注册者须年满 18 周岁
- 使用自有顶级域名（推荐 `.com`），免费二级域名通过率极低
- 能修改网站 HTML 源代码以放置 AdSense 验证代码

## 内容合规红线（一票否决）

网站**绝对不允许**出现以下内容：
- 色情、赌博、暴力血腥
- 盗版资源（软件下载、影视爬取、小说搬运）
- 仇恨言论、欺诈信息、虚假医疗
- AI 批量洗稿 / 伪原创 / 关键词农场
- 无意义资讯采集、壁纸采集
- YMYL（Your Money Your Life）低质量内容

## 2026 审核核心标准

Google 不反 AI，反的是**没有人工作价值的规模化 AI 内容**。

**✅ 可以的**：AI 辅助写作、AI 搭框架、AI 做整理 + 人工修改 + 真实经验
**❌ 不可以的**：一键生成数百篇、无人工修改、无真实信息、模板化结构

内容必须体现：**真实经验、真实观点、真实信息增量**。
- 实测教程、对比评测、自己拍的图、自己测试的数据 > 泛泛而谈
- 深度分析、行业经验 > 全网重复内容、空洞百科

## 必备页面

网站必须有：
- **About**（关于我们）
- **Contact**（联系方式）
- **Privacy Policy**（隐私政策）— 说明如何处理用户数据
- **Terms / Disclaimer**（免责声明）

缺少这些页面会被判定为临时垃圾站。

## 技术要求

- **HTTPS**：必须配置 SSL 证书
- **移动端适配**：移动端体验权重极高，必须正常显示
- **加载速度**：避免大图、花哨动画、过多广告插件；Core Web Vitals 不能太差

## 过审公式

> **明确领域 + 20 篇以上真实原创内容（每篇 800–1500+ 字）+ 完整的网站结构（About/Contact/Privacy/Terms）+ 移动端体验良好 + HTTPS + Google 正常收录 + 看起来像长期运营的站**

### 通过率较高的类型
工具站（带实用功能）、深度教程站、垂直行业站、真实测评站、数据分析站、AI 工具教程站、小众专业知识站

### 容易被拒的类型
影视下载、壁纸采集、小说搬运、GPT 批量站、无意义资讯站、医疗/YMYL 低质量站、"什么都写"的杂站

## 部署经验总结

### GitHub 推送注意事项
1. **修改后务必提交**：用 Edit 改完文件后要 `git add` + `git commit`，多次修改之间及时提交避免漏改
2. **国内网络问题**：直连 GitHub 经常连接重置。推送前加代理：
   ```bash
   https_proxy=http://127.0.0.1:7897 git push
   ```
3. **提交前检查**：`git status` 确认所有改过的文件都在暂存区
4. **提交后验证**：去 GitHub 仓库页面刷新确认文件已更新

### Cloudflare Pages 部署注意事项
1. **纯静态站点不要用 Worker 模式**：Build command 必须清空或用 `true`，Build output directory 填 `.`
2. **wrangler.jsonc 配置文件**：在项目根目录放此文件，Cloudflare 自动识别为静态站点
3. **构建配置修改**：在 Cloudflare Dashboard → 项目 Settings → Build configuration 中修改
4. **部署触发**：推送到 GitHub 后 Cloudflare 自动触发部署

### 技术注意事项

1. **不要用 CDN 动态加载 Tailwind**：`https://cdn.tailwindcss.com` 会做版本重定向，Google 爬虫会报"重定向错误"。改用预编译的 CSS 文件或直接写内联样式来解决
